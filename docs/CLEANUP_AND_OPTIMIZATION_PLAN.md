**修复内容**：

   - 新增参数 `min_cross_distance = 0.0005`（0.05%）

   - 在 `_process_trading_signal` 开头检查持仓
   - 单一方向持仓上限为2手
   - 已有同方向持仓达到上限时跳过信号

### 2025-12-30 MaRsiComboStrategy 重大改进

**改进内容：两阶段时间确认机制**

**问题背景**：
- 原来的交叉幅度计算改为"总穿越距离"后，阈值要求太高
- 黄金价格波动缓慢，很多正常交叉也达不到阈值
- 但降低阈值又会导致假突破增多

**解决方案：两阶段时间确认**

```
第一阶段（检测交叉）：
  - 发现MA5穿越MA20 → 记录为"待确认"状态
  - 记录交叉时的MA差值

第二阶段（等待确认，默认2根K线）：
  - 每根新K线检查MA5是否仍在正确一侧
  - 假突破：MA5回撤到错误一侧 → 取消待确认，不发信号
  - 真突破：差值保持50%以上 + 价格确认 → 发出交叉信号
```

**新增参数**：
- `cross_confirm_bars = 2`：确认所需K线数
- `min_cross_distance = 0.0002`：最小穿越幅度（降至0.02%，主要靠时间确认）

**优势**：
- 假突破通常1-2根K线内回撤，自动被过滤
- 真趋势转折会保持或扩大差值，能正常触发
- 不需要设置高幅度阈值，对缓慢行情友好
