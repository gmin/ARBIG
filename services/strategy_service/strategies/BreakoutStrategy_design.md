# BreakoutStrategy 设计文档

## 1. 策略概述

**布林带突破策略** 是专门针对上期所黄金期货设计的突破交易策略，通过布林带通道识别价格突破，结合RSI过滤和N根K线回踩确认机制，在趋势启动阶段入场。

### 1.1 适用市场
- ✅ 上期所黄金期货（au主力合约）
- ✅ 趋势启动阶段，价格突破重要通道边界
- ✅ 波动性放大行情
- ⚠️ 震荡市假突破较多，需谨慎使用

### 1.2 风险特征
- 📊 **风险等级**：中高
- 💰 **资金要求**：适中（每手约5-10万保证金）
- ⏱️ **持仓周期**：分钟级到小时级
- 📈 **预期收益**：进攻型，追求趋势启动阶段的大幅波动

---

## 2. 核心交易逻辑

### 2.1 信号生成流程

```
K线数据 → 更新ArrayManager → 计算布林带/RSI/ATR
    → 检测价格是否突破布林带上下轨
    → 突破幅度 >= 0.2%？ → 进入待确认状态
    → 等待N根K线回踩确认 → RSI过滤 → 生成交易信号
```

### 2.2 技术指标组合

| 指标 | 参数 | 用途 |
|------|------|------|
| 布林带 | BB(20, 2.0) | 识别价格通道，检测突破 |
| RSI14 | 14周期 | 过滤追高杀低（上突破RSI<70，下突破RSI>30） |
| ATR14 | 14周期 | 波动性参考 |

### 2.3 信号生成条件

| 信号类型 | 条件 |
|----------|------|
| 🟢 **买入信号** | 收盘价突破布林带上轨（强度≥0.2%）+ 2根K线回踩确认 + RSI<70 |
| 🔴 **卖出信号** | 收盘价突破布林带下轨（强度≥0.2%）+ 2根K线回踩确认 + RSI>30 |
| ⏸️ **观望** | 无突破 / 突破幅度不足 / RSI极端区域 / 回踩过深（假突破） |

## 3. 突破确认机制

### 3.1 问题背景

简单的布林带突破策略面临**假突破**问题：价格短暂突破后立即回落，导致追高被套。

### 3.2 解决方案：N根K线回踩确认 + 容忍度过滤

```
┌──────────────────────────────────────────────────────────────┐
│  第一阶段：检测突破                                            │
│  ────────────────────                                        │
│  收盘价突破布林带上/下轨                                        │
│  突破强度 = |收盘价 - 轨道| / 轨道 >= 0.2%                      │
│        ↓                                                     │
│  进入"待确认"状态，记录：                                       │
│  • 突破价格（收盘价）                                          │
│  • 突破的轨道值（上轨/下轨）                                    │
│  • 突破强度                                                   │
│  • 初始极值（上突破=最高价，下突破=最低价）                       │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│  第二阶段：等待2根K线确认                                       │
│  ────────────────────────                                    │
│  每根K线更新极值                                               │
│                                                              │
│  回踩检查：                                                   │
│  • 上突破：收盘价跌回上轨内的幅度 <= 容忍度 × 突破强度？          │
│  • 下突破：收盘价涨回下轨内的幅度 <= 容忍度 × 突破强度？          │
│                                                              │
│  达到确认K线数时：                                              │
│  • 回踩在容忍范围内 → ✅ 确认突破，保存强度到confirmed变量        │
│  • 回踩过深 → ❌ 假突破，取消信号                               │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 突破强度计算

```python
# 上突破
strength = (close - upper_band) / upper_band  # 正数，越大突破越强

# 下突破
strength = (lower_band - close) / lower_band  # 正数，越大突破越强
```

### 3.4 回踩容忍度

```python
# 回踩幅度
pullback = (band - close) / band  # 收盘价跌回轨道内的程度

# 容忍条件
breakthrough_held = pullback <= pullback_tolerance × min_breakout_strength
# 默认：pullback <= 0.5 × 0.002 = 0.1%
```

### 3.5 相关参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `breakout_confirm_bars` | 2 | 确认所需K线数 |
| `min_breakout_strength` | 0.002 (0.2%) | 最小突破幅度阈值 |
| `pullback_tolerance` | 0.5 | 回踩容忍度：回踩不超过突破幅度的50% |

## 4. 持仓管理逻辑

### 4.1 信号处理流程

**上突破信号（BUY）处理流程**：
```
BUY信号
│
├── 1. 检查空头持仓（反向持仓处理）
│   └── 有空头 → ✅ 强制平仓（不区分盈亏）
│
└── 2. 检查是否开多
    ├── 多头达上限(max_position) → ⚠️ 不开仓
    ├── 无多头持仓 → ✅ 直接开仓
    └── 有多头 → ✅ 加仓
```

**下突破信号（SELL）处理流程**：
```
SELL信号
│
├── 1. 检查多头持仓（反向持仓处理）
│   └── 有多头 → ✅ 强制平仓（不区分盈亏）
│
└── 2. 检查是否开空
    ├── 空头达上限(max_position) → ⚠️ 不开仓
    ├── 无空头持仓 → ✅ 直接开仓
    └── 有空头 → ✅ 加仓
```

### 4.2 与 MaRsiComboStrategy 的差异

| 特性 | BreakoutStrategy | MaRsiComboStrategy |
|------|-----------------|-------------------|
| 反向信号处理 | 强制平仓（不区分盈亏） | 盈利平仓，亏损保留 |
| 加仓条件 | 只要不超上限就加仓 | 需亏损达ATR阈值 |
| 对锁 | ❌ 不允许 | ✅ 允许 |

## 5. 风险控制

### 5.1 风控参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `long_stop_loss_pct` | 0.8% | 多单止损比例 |
| `long_take_profit_pct` | 5.0% | 多单止盈比例 |
| `short_stop_loss_pct` | 0.8% | 空单止损比例 |
| `short_take_profit_pct` | 5.0% | 空单止盈比例 |
| `max_position` | 3手 | 单方向最大持仓 |

### 5.2 多空独立风控

与 MaRsiComboStrategy 的统一止损止盈不同，BreakoutStrategy 支持**多空独立参数**：

```python
# 多单风控
long_pnl_pct = (current_price - long_price) / long_price
if long_pnl_pct <= -long_stop_loss_pct:  # 止损
if long_pnl_pct >= long_take_profit_pct:  # 止盈

# 空单风控
short_pnl_pct = (short_price - current_price) / short_price
if short_pnl_pct <= -short_stop_loss_pct:  # 止损
if short_pnl_pct >= short_take_profit_pct:  # 止盈
```

### 5.3 风控触发层级

- **K线级别**（`on_bar_impl`）：每根K线收盘时检查
- **Tick级别**（`on_tick_impl`）：实时价格检查，更及时

## 6. 系统架构

### 6.1 双层处理架构

```
┌─────────────────────────────────────────┐
│  K线级别 (on_bar_impl)                  │
│  ─────────────────────                  │
│  • 计算布林带/RSI/ATR                    │
│  • 突破检测 + N根K线确认                  │
│  • 风控检查                              │
│  • 信号生成 + 交易执行                    │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Tick级别 (on_tick_impl)                │
│  ─────────────────────                  │
│  • 实时止损止盈检查                      │
└─────────────────────────────────────────┘
```

### 6.2 框架集成

继承 `ARBIGCtaTemplate`，遵循以下规范：
- 使用 `on_bar_impl` / `on_tick_impl` / `on_trade_impl`，**不覆盖**父类的 `on_bar` / `on_tick` / `on_trade`
- 持仓变量（`long_pos`/`short_pos`/`long_price`/`short_price`）由**父类 `on_trade`** 统一更新
- `on_trade_impl` 只负责日志记录和文件持久化，不手动修改持仓变量

### 6.3 持仓持久化

```
data/real_positions_{strategy_name}_{symbol}.json
```

- **保存时机**：`on_trade_impl`（成交回调）和 `on_stop`（策略停止）
- **加载时机**：`__init__`（策略初始化）
- **数据结构**：与 MaRsiComboStrategy 相同

### 6.4 智能平仓（上期所特殊处理）

与 MaRsiComboStrategy 相同：查询今昨仓 → 优先平昨仓 → 再平今仓

## 7. 参数调优建议

### 7.1 黄金期货推荐参数

```python
{
    'bollinger_period': 20,
    'bollinger_std': 2.0,
    'rsi_period': 14,
    'rsi_overbought': 70,
    'rsi_oversold': 30,
    'breakout_confirm_bars': 2,
    'min_breakout_strength': 0.002,   # 0.2%
    'pullback_tolerance': 0.5,
    'long_stop_loss_pct': 0.008,      # 0.8%
    'long_take_profit_pct': 0.05,     # 5.0%
    'short_stop_loss_pct': 0.008,
    'short_take_profit_pct': 0.05,
    'trade_volume': 1,
    'max_position': 3,
}
```

### 7.2 参数调整指南

| 场景 | 调整建议 |
|------|----------|
| 假突破太多 | 增大 `breakout_confirm_bars` 到 3，降低 `pullback_tolerance` 到 0.3 |
| 错过突破 | 减小 `min_breakout_strength` 到 0.001，增大 `pullback_tolerance` 到 0.7 |
| 追高杀低 | 收紧 RSI 过滤：`rsi_overbought=65`, `rsi_oversold=35` |
| 波动大行情 | 增大止损 `stop_loss_pct`，增大 `min_breakout_strength` |

| 标识 | 含义 |
|------|------|
| 🔔 [突破检测] | 检测到价格突破布林带，进入待确认期 |
