# MeanReversionStrategy 设计文档

## 1. 策略概述

**均值回归策略** 是专门针对上期所黄金期货设计的震荡市交易策略，利用布林带+RSI双重确认在价格偏离均值时入场，以布林带中轨（均值）为止盈目标。

### 1.1 适用市场
- ✅ 上期所黄金期货（au主力合约）
- ✅ 震荡整理行情，价格在布林带通道内来回波动
- ✅ 布林带带宽收窄（低波动率）时效果最佳
- ⚠️ 趋势行情中会被止损（策略有带宽过滤保护）
- ❌ 不适合单边趋势行情

### 1.2 风险特征
- 📊 **风险等级**：中低
- 💰 **资金要求**：适中（每手约5-10万保证金）
- ⏱️ **持仓周期**：分钟级到小时级（快进快出）
- 📈 **预期收益**：稳健型，追求小幅高频的均值回归利润

### 1.3 与趋势/突破策略的本质区别

| | 趋势/突破 | 均值回归 |
|---|----------|---------|
| **交易方向** | 顺势（涨→买，跌→卖） | 逆势（跌→买，涨→卖） |
| **入场点** | 趋势形成/突破时 | 价格偏离均值极端时 |
| **止盈目标** | 趋势延续（大利润） | 回归均值（小利润） |
| **适用行情** | 趋势市 | 震荡市 |
| **持仓时间** | 较长 | 较短 |

## 2. 核心交易逻辑

### 2.1 信号生成流程

```
K线数据 → 更新ArrayManager → 计算布林带/RSI/ATR/带宽
    → 带宽过滤：带宽 > 3%？ → 是：趋势市，不交易
    → 否：检测回归信号（布林带触及 + RSI极端）
    → N根K线确认 → 生成交易信号
```

### 2.2 技术指标组合

| 指标 | 参数 | 用途 |
|------|------|------|
| 布林带 | BB(20, 2.0) | 识别价格通道边界 |
| 布林带带宽 | (上轨-下轨)/中轨 | 波动率过滤（>3%不交易） |
| RSI14 | 14周期 | 超买超卖确认（超买>75，超卖<25） |
| ATR14 | 14周期 | 波动性参考 |

### 2.3 信号生成条件

| 信号类型 | 条件 |
|----------|------|
| 🟢 **做多信号** | 价格在下轨0.1%内 + RSI<25 + 带宽<3% + 连续2根K线确认 |
| 🔴 **做空信号** | 价格在上轨0.1%内 + RSI>75 + 带宽<3% + 连续2根K线确认 |
| ⏸️ **观望** | 带宽>3%（趋势市）/ 条件不满足 / 确认失败 |

**注意**：RSI阈值（25/75）比 MaRsiComboStrategy（30/70）更极端，因为均值回归需要更强的超买超卖信号。

## 3. 三重过滤机制

### 3.1 第一层：带宽过滤

```python
bandwidth = (upper - lower) / middle

if bandwidth > max_bandwidth_pct (3%):
    # 趋势市 → 不做均值回归
    # 取消任何待确认信号
```

**设计思路**：布林带带宽反映市场波动率。带宽大说明趋势性强，均值回归容易亏损。

### 3.2 第二层：布林带 + RSI 双重确认

```python
# 做多条件
lower_distance = (price - lower_band) / lower_band
if lower_distance <= 0.001 and RSI < 25:  # 触及下轨 + RSI超卖
    signal = "LONG"

# 做空条件
upper_distance = (upper_band - price) / upper_band
if upper_distance <= 0.001 and RSI > 75:  # 触及上轨 + RSI超买
    signal = "SHORT"
```

### 3.3 第三层：N根K线确认

```
检测到回归信号
├── 首次检测 → 记录方向、价格、RSI，计数=1
├── 第2根K线仍满足条件 → 计数=2 → ✅ 确认信号
├── 第2根K线条件不满足 → ❌ 重置
└── 方向反转 → 重置，开始新方向确认
```

### 3.4 相关参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_bandwidth_pct` | 0.03 (3%) | 最大带宽，超过不交易 |
| `min_band_touch_pct` | 0.001 (0.1%) | 价格触及轨道的距离阈值 |
| `reversion_confirm_bars` | 2 | 确认所需K线数 |
| `rsi_overbought` | 75 | RSI超买阈值 |
| `rsi_oversold` | 25 | RSI超卖阈值 |

## 4. 风险控制（核心特色）

### 4.1 中轨止盈（均值回归本质）

**核心理念**：均值回归交易的目标就是价格回到均值。布林带中轨 = 均值。

```python
# 多仓中轨止盈
if use_middle_band_tp and current_price >= current_middle:
    平仓  # 价格回到中轨，目标达成

# 空仓中轨止盈
if use_middle_band_tp and current_price <= current_middle:
    平仓  # 价格回到中轨，目标达成
```

**vs 固定止盈**：
- 中轨止盈：**动态目标**，随市场波动调整
- 固定止盈（2%）：**备用方案**，中轨止盈未触发时的兜底

### 4.2 风控参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `long_stop_loss_pct` | 0.6% | 多单止损（比突破策略更紧） |
| `long_take_profit_pct` | 2.0% | 多单固定止盈（备用） |
| `short_stop_loss_pct` | 0.6% | 空单止损 |
| `short_take_profit_pct` | 2.0% | 空单固定止盈（备用） |
| `use_middle_band_tp` | True | 是否启用中轨止盈 |
| `max_position` | 3手 | 单方向最大持仓 |

### 4.3 风控检查优先级

```
on_bar_impl 执行顺序：
    1. 计算指标
    2. ⚡ 先检查风控（止损/中轨止盈/固定止盈）
    3. 风控触发？ → 跳过信号生成
    4. 检查交易时间
    5. 生成交易信号
```

**设计思路**：风控优先于信号，保护已有持仓。

### 4.4 止损 vs 止盈优先级

```
多仓检查顺序：
    止损 → 中轨止盈 → 固定止盈
    （止损最优先，中轨止盈次之，固定止盈兜底）
```

## 5. 持仓管理逻辑

### 5.1 信号处理流程

**做多信号处理**：
```
LONG信号
│
├── 1. 有空头？ → 先平空（反向信号强制平仓）
│
└── 2. 开多仓
    ├── 多头达上限 → ⚠️ 不开仓
    └── 未达上限 → ✅ 开仓
```

### 5.2 与 MaRsiComboStrategy 的差异

| 特性 | MeanReversionStrategy | MaRsiComboStrategy |
|------|----------------------|-------------------|
| 反向信号处理 | 强制平仓（不区分盈亏） | 盈利平仓，亏损保留 |
| 加仓机制 | 无（信号触发即开仓） | ATR动态阈值加仓 |
| 对锁 | ❌ 不允许 | ✅ 允许 |
| 止盈方式 | 中轨止盈（动态） | 固定百分比 |
| 止损幅度 | 0.6%（更紧） | 0.6%（相同） |

**不允许对锁的原因**：均值回归是逆势交易，如果市场转为趋势，应该快速止损离场，而不是对锁。

## 6. 系统架构

### 6.1 双层处理架构

```
┌─────────────────────────────────────────┐
│  K线级别 (on_bar_impl)                  │
│  ─────────────────────                  │
│  • 计算布林带/RSI/ATR/带宽               │
│  • 风控检查（优先）                      │
│  • 带宽过滤 → 三重确认 → 信号生成         │
│  • 交易执行                              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Tick级别 (on_tick_impl)                │
│  ─────────────────────                  │
│  • 实时止损/中轨止盈检查                  │
└─────────────────────────────────────────┘
```

### 6.2 框架集成

继承 `ARBIGCtaTemplate`，遵循以下规范：
- 使用 `on_bar_impl` / `on_tick_impl` / `on_trade_impl`
- 持仓变量由**父类 `on_trade`** 统一更新，子类不手动修改
- `on_trade_impl` 只负责日志记录和文件持久化

### 6.3 持仓持久化

与 MaRsiComboStrategy 和 BreakoutStrategy 相同：
```
data/real_positions_{strategy_name}_{symbol}.json
```

## 7. 参数调优建议

### 7.1 黄金期货推荐参数

```python
{
    'bollinger_period': 20,
    'bollinger_std': 2.0,
    'rsi_period': 14,
    'rsi_overbought': 75,
    'rsi_oversold': 25,
    'reversion_confirm_bars': 2,
    'max_bandwidth_pct': 0.03,       # 3%
    'min_band_touch_pct': 0.001,     # 0.1%
    'use_middle_band_tp': True,
    'long_stop_loss_pct': 0.006,     # 0.6%
    'long_take_profit_pct': 0.02,    # 2.0%
    'short_stop_loss_pct': 0.006,
    'short_take_profit_pct': 0.02,
    'trade_volume': 1,
    'max_position': 3,
}
```

### 7.2 参数调整指南

| 场景 | 调整建议 |
|------|----------|
| 信号太少 | 放宽 `max_bandwidth_pct` 到 4%，放宽 RSI 到 30/70 |
| 假信号多 | 增大 `reversion_confirm_bars` 到 3，收紧 RSI 到 20/80 |
| 止盈太早 | 关闭 `use_middle_band_tp`，使用固定止盈 |
| 止损太频繁 | 增大 `stop_loss_pct` 到 1.0% |
| 震荡市效果差 | 减小 `min_band_touch_pct` 到 0.002 |

## 8. 日志输出说明

### 8.1 关键日志标识

| 标识 | 含义 |
|------|------|
| 📊 [带宽过滤] | 带宽过大，不做交易 |
| 📊 [回归检测] | 检测到回归信号，开始确认 |
| 📊 [回归确认] | 确认中，显示确认进度 |
| 📊 [回归取消] | 条件不再满足，重置 |
| 🛑 [多仓止损] / [空仓止损] | 止损触发 |
| 🎯 [多仓中轨止盈] / [空仓中轨止盈] | 中轨止盈触发 |
| ✅ [多仓止盈] / [空仓止盈] | 固定止盈触发 |
| 📈 [开多] / 📉 [开空] | 开仓执行 |
| 🔄 [反向平仓] | 反向信号平仓 |
